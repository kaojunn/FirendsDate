# GitHub Pages 发布说明

当前项目已经准备好了网页入口文件：

- `index.html`
- `展商导航.html`

其中：

- `index.html` 是 GitHub Pages 默认入口
- 打开后会自动跳转到 `展商导航.html`

## 一、发布前建议

如果你刚修改过展商资料，先重新生成导航页：

```bash
python3 生成展商导航.py
```

然后提交并推送：

```bash
git add .
git commit -m "更新展商导航发布文件"
git push
```

## 二、在 GitHub 开启 Pages

进入仓库网页后按以下步骤操作：

1. 打开仓库 `FirendsDate`
2. 进入 `Settings`
3. 点击左侧 `Pages`
4. 在 `Build and deployment` 中选择：
   - `Source`: `Deploy from a branch`
   - `Branch`: `main`
   - `Folder`: `/ (root)`
5. 点击保存

## 三、发布后访问方式

GitHub Pages 启用后，GitHub 会生成一个站点地址。  
默认入口就是：

- `index.html`

因此访问 Pages 地址时，会自动跳转到：

- `展商导航.html`

## 四、后续更新方式

后续只要按这个流程即可：

1. 修改展商文件夹中的 `介绍卡.md`
2. 运行 `python3 生成展商导航.py`
3. `git add .`
4. `git commit`
5. `git push`

推送完成后，GitHub Pages 会自动更新网页内容。

## 五、注意事项

- 不要手动编辑 `展商导航.html` 里的展商数据
- 展商资料应优先修改各展商目录里的 `介绍卡.md`
- 如果网页没有立刻更新，通常等待几十秒到几分钟即可
