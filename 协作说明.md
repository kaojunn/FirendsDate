# 协作说明

本文档用于说明团队成员如何在这个展商资料库中协作更新资料，避免多人同时维护时出现混乱。

## 先看这 3 条规则

1. 不要手动修改 `展商导航.html`
2. 优先修改各展商文件夹中的 `介绍卡.md`
3. 修改完成后，运行 `python3 生成展商导航.py`

## 一、你应该改哪里

### 如果你要补公司介绍

请修改对应展商文件夹下的：

- `介绍卡.md`

### 如果你要补联系人和沟通信息

目前也请先写进：

- `介绍卡.md`

后续如果资料越来越多，可以再为每家新增：

- `联系方式.md`
- `跟进记录.md`

### 如果你要修改页面展示逻辑

请修改：

- `生成展商导航.py`

不要直接编辑：

- `展商导航.html`

## 二、标准更新流程

### 1. 先同步最新内容

如果已经接入远程仓库，先执行：

```bash
git pull
```

### 2. 编辑资料

修改对应展商的：

- `介绍卡.md`

### 3. 重新生成 HTML

```bash
python3 生成展商导航.py
```

### 4. 本地检查

确认：

- `展商导航.html` 能正常打开
- 新修改的展商能在页面中搜到
- 详情区展示正常

### 5. 提交修改

```bash
git add .
git commit -m "更新某某展商资料"
git push
```

## 三、建议填写规范

建议所有 `介绍卡.md` 尽量统一写法，至少包含以下内容：

- 来源
- 分区
- 展位
- 简介
- 公开链接
- 已掌握联系人
- 后续可补充

## 四、推荐提交信息

推荐使用这种风格：

- `补充 Story 联系方式`
- `更新 OpenSpot AI 业务介绍`
- `新增 5 家展商链接`
- `优化 展商导航 页面筛选逻辑`

## 五、冲突避免建议

为了减少多人编辑冲突：

- 一次尽量只改自己负责的展商文件夹
- 不要同时大规模改很多无关文件
- 提交前先重新生成一次 `展商导航.html`
- 推送前先拉取最新代码

## 六、发布建议

如果后续要给团队统一浏览，推荐把仓库接到：

- GitHub Pages
- Gitee Pages
- 公司内网静态站点

这样同事可以直接通过网址访问 `展商导航.html`。
